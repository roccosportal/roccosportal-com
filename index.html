Projects

simpubly

This is the scenario: You own a webspace where you only can execute php files. You can not install git or other publishing software. Still you want to be able to publish the newest version from your github project to your webspace without manually copy and paste the files. This php script allows you to ‘publish’ your php/html/js project by calling an url. It has some basic features like making a backup and ignoring files. Have a look.

less-watchr

The less npm package does not come with a file watcher that can recompile less files on a file change event. This node package closes the gap. It is an easy to use command line tool and comes with a lot of customization features. For example it allows you to recompile multiple less files when one file was changed. This feature was necessary in one of my bigger projects where we had seperated less folders that had to be recombiled when a global less file was changed.

Pvik

Pvik is just another php5 framework and is more an expirement and learning project than an attempt to create a serious framework in the row with other big frameworks like Symfony, ZendFramework and Fatfree. It follows the MVC concepts and provides a database abstraction layer which supports so far MySQL and MSSQL. Even Pvik is a one person project it has a wiki to help you building websites with it. A boilerplate is available, you can start boiling here. The source code is located on Github. Please feel free to contribute.
